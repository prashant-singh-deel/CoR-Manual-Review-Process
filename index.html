a. Client / Contractor Legal Entity

  • Contractor must NOT be hired in same country as the Legal Entity
  • If same country → escalate to Compliance (Jira)
  • If California (USA) → contract must be cancelled; notify @James Moore & @Maciej Dalkowski
  • Non-USA same-country: no Legal ticket needed
  • Check Assessment tab for pre-filled questionnaire

b. Job Title

  • Title cannot be C-level (Director and above)
  • If C-level → escalate to Compliance first, then Legal (US)
  • Check Assessment tab for pre-filled questionnaire
  • Edit job description if not aligned with requirements

c. Special Clauses

  • No Special Clauses allowed in the contract
  • If only Special Clauses triggered → Legal Ops Jira
  • If Special Clauses + other triggers → escalate both to Compliance & Legal
  • Check Assessment tab for pre-filled questionnaire

d. Scope of Work (SOW)

  • Client or entity name must NOT appear in job title or SOW
  • No prohibited keywords (cannabis, weapons, etc. — see ToS §7)
  • Non-English SOW: translate to English (Google Translate / screenshot) and add copy to WB note
  • Flag any issues to CSM

e. Additional Documents

  • No Additional Documents should be attached to the contract
  • If found: flag to CSM to remove

f. Stock Options

  • No stock options may be added to the contract
  • If found: flag to CSM to delete
  • Notify @James Moore if stock options are present

g. Equipment

  • Only laptops and mobile phones are permitted
  • Any other equipment type → flag to CSM to remove

h. Start Date

  • Start date can be a max of 7 days in the past
  • Any start date within last 7 days is acceptable
  • If beyond 7 days past → flag to CSM to update to today's date
  • No escalation to Compliance/Legal needed
💡 False Positive Word Matches: If a word in the SOW matches the client org name (e.g. "imple" flagged for "Ximple"), you can add the term as a compliance exception in the client's MSA → Agreements → COR Agreement → Compliance Exemptions → Special keywords. Then confirm no blocker in the Slack thread.

🔴 Escalation to Compliance Team

Use when: same-country legal entity (non-CA), C-level job title, or special clauses are present.

  1. Open the Privacy and Compliance Service Desk (Jira)
  2. Summary: Brief reason for escalation (e.g. "Contractor hired in same country as legal entity")
  3. Attachment: Attach the filled questionnaire from the Assessment tab
  4. Share With: Select Deel
  5. "What is your request related to?" → Select COR
  6. Link to admin profile: Copy URL from WB task (e.g. admin.deel.network/contracts/xxxxx/profile)
  7. Admin Organization Profile link: Right-click the client org name in Admin to copy
  8. "Which Deel product?" → Select COR
  9. Request Description: Full detail on reason for escalation. Include questionnaire answers for US contracts.
  10. "ZD Ticket?" → Select No
  11. Paste Jira ticket link into the Jira section in Workbench
  12. Change WB status to Escalated
Search for existing Jira tickets first using the GSCMB-5 template before creating a new one.

⚖️ Escalation to Legal Team

Use when: special clauses present (after Compliance review), California US cases, or cases requiring legal sign-off.

  1. Open the Legal Operations Service Desk (Jira)
  2. Summary: Brief reason (e.g. "Special Clauses present in COR contract")
  3. Link to admin profile: WB task URL
  4. Admin Organization Profile link: Right-click client org name
  5. Specific contractor (yes/no): Yes
  6. Admin contract link: Same as admin profile link
  7. Contractor Work Country: Check employment country in Admin
  8. Client has signed MSA?: Go to Contract Agreement section → click MSA → verify client signature on last page
  9. Counterparty name: Click client name → scroll to Org Admin section → copy name
  10. Counterparty email: Copy email below Org Admin name
  11. Description: Full detail on escalation reason
  12. ZD ticket (yes/no): No (in most cases)
  13. Paste Jira link into WB → set status to Escalated

How to Request a Compliance Exemption (3-Step Process)

📋 Step 1: Complete the Compliance Questionnaire

Provides the full business context behind the exemption request.

  • Make a copy of the Deel COR Questionnaire (Google Sheets)
  • Complete all relevant fields explaining the business case
  • This questionnaire will be used as the Supporting Document in Step 2
The questionnaire may already be pre-filled during CoR contract creation — check the Assessment tab on the contract first. If filled with "N/A", ask the CSM to have the Client fill it out properly.

⚖️ Step 2: Legal Review via COHD Form

Raise a COR Compliance Exemption Request ticket to Legal.

  • Client / Org ID: Admin link to the organisation
  • Problem Statement: Description of why the default requirement cannot be met
  • Requested Change: Specific exemption requested (e.g. "Change Job Title to Director level")
  • Supporting Document: Attach the completed COR Questionnaire from Step 1
Ticket is submitted via the COHD form in the Ops Workbench Request Portal.

🔧 Step 3: Backend Change in Product (Post-Legal Approval)

After Legal approves the exemption, raise a request to the Techops team.

  • Use the Internal Webform or the COHD form to request the backend change
  • Exemptions are applied at the organizational level — they apply to all contracts for that client org
  • For word-match false positives (e.g. SOW word matching org name): add the term to the client's MSA → Agreements → COR Agreement → Compliance Exemptions → Special Keywords

FAQs

❓ Can interns be hired via COR?

No. Interns are required to receive direct guidance and supervision, which is inconsistent with the independent contractor model. Interns should not be hired via COR.

❓ Can a COR contractor be hired in a corp-to-corp relationship?

Yes. An independent contractor can be hired through Deel as an individual or through their own entity (corp-to-corp). Both models are supported under COR.
